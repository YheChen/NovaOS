# NovaOS v1.0.0

The first public release of NovaOS - a browser-based, deterministic
operating-systems laboratory that takes a program from source to execution and
makes every layer inspectable.

## Highlights

- **End-to-end pipeline.** Write Toy C, compile it through a real toolchain
  (lexer → parser → type checker → IR → optimizer → NovaASM → assembler →
  bytecode), and run it on a custom 32-bit virtual machine driven by a
  microkernel.
- **Transparent at every layer.** Inspect tokens, AST, IR, optimized IR,
  assembly, and bytecode; watch registers, memory, the scheduler, and syscalls
  as the program runs.
- **Time-travel debugger.** Step by instruction or source line, step into/over/
  out, set line/instruction/conditional/exception/memory breakpoints, watch
  expressions (no `eval`), read the reconstructed call stack, and rewind through
  a deterministic timeline.
- **Workspace UI.** A Vite + React SPA wires the editor, compile-stage inspector,
  run output, and a live debugger panel to the real domain packages - no mocked
  state.

## Toy C v1 language

`int` / `bool` / `void`, variables, assignment, arithmetic, comparison, logical
operators, `if`/`else`, `while`, functions, parameters, `return`, recursion, and
`print`.

## Engineering

- ~250 automated tests across unit, integration, golden, and deterministic-replay
  suites, plus a Playwright E2E flow against the built app.
- Strict TypeScript, enforced package boundaries, an acyclic dependency graph,
  and a determinism gate - all checked in CI.
- Decisions recorded as ADRs; the build plan and quality gates live in
  `docs/orchestration/`.

## Known limitations (documented, by design for v1)

- Toy C integer literals are 0-65535 (16-bit immediate load); arithmetic is
  unsigned 32-bit at the print boundary.
- `&&` / `||` evaluate both operands (non-short-circuit); operands have no side
  effects in practice.
- The editor is a plain text area (Monaco integration is a planned follow-up).
- Filesystem persistence is in-memory per session (IndexedDB persistence planned).
- Time-travel replay reconstructs state by replaying from the start.
